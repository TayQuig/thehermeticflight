## Summary

The Hermetic Flight has strong content foundations — proper heading hierarchy,
adequate word counts, working robots.txt and sitemap, no mixed content, and
valid JSON-LD on the homepage. However, the site is critically missing Open
Graph and Twitter Card meta tags on all pages, which prevents social media
previews entirely. Canonical URLs inconsistently omit the `www` prefix, and
structured data (JSON-LD) is absent from all pages except the homepage.

## Critical Issues (Fix Now)

### 1. Open Graph tags missing on ALL pages
**Severity:** Critical
**Pages:** All 6
**Impact:** Social media shares (Facebook, LinkedIn, Slack) show no preview image, title, or description. For a Kickstarter pre-launch campaign relying on social sharing, this eliminates the viral loop entirely.
**Evidence:** `og:title`, `og:description`, `og:image`, `og:url` all return `null` on every page.
**Fix:** Add OG meta tags to Layout.astro. **[Fixed in PR]** — the feature branch adds full OG tags with per-page customization.

### 2. Twitter Card tags missing on ALL pages
**Severity:** Critical
**Pages:** All 6
**Impact:** X/Twitter shares show plain text links with no card preview. Similar viral loop impact as OG tags.
**Evidence:** `twitter:card`, `twitter:title`, `twitter:description` all return `null`.
**Fix:** Add Twitter Card meta tags to Layout.astro. **[Fixed in PR]**

### 3. JSON-LD structured data absent from 5 of 6 pages
**Severity:** Critical
**Pages:** Quiz, FAQ, Blog Index, Thank You, Blog Post
**Impact:** Search engines cannot extract structured data for rich snippets. Blog posts missing Article schema means no rich results for publication date, author, or headline. FAQ page could benefit from FAQPage schema.
**Evidence:** Only the homepage has `<script type="application/ld+json">` (Product schema). All other pages return `null`.
**Fix:** Add Article schema to blog posts (datePublished, author, headline). Consider FAQPage schema for FAQ.

### 4. Missing H1 on Quiz and Thank You pages
**Severity:** Critical
**Pages:** Quiz (`/quiz`), Thank You (`/thank-you`)
**Impact:** H1 is the primary heading signal for search engines. Missing H1 weakens page topic identification.
**Fix:** Add visible or visually-hidden H1 elements to both pages.

### 5. Blog post titles exceed 60-character limit
**Severity:** Critical
**Page:** Blog Post (Welcome to the Flight)
**Evidence:** Title is 93 characters: "Welcome to The Hermetic Flight: Where Aerial Arts Meet Tarot Divination | The Hermetic Flight" — Google truncates at ~60 characters.
**Fix:** Shorten the title tag or use a shorter version for the `<title>` element while keeping the full title in the H1.

### 6. Blog post meta descriptions exceed 160 characters
**Severity:** Critical
**Page:** Blog Post (Welcome to the Flight)
**Evidence:** Meta description is 211 characters — search engines truncate at ~160.
**Fix:** Trim to 150-160 characters while preserving the primary keyword and compelling call to read.

## Warnings (Fix Soon)

### 1. Canonical URLs missing `www` prefix
**Pages:** All 6
**Evidence:** Canonical is `https://thehermeticflight.com/...` but the live site serves at `https://www.thehermeticflight.com/...`. This can cause search engines to index both www and non-www versions as separate pages.
**Fix:** Update canonical URLs to include `www`. **[Fixed in PR]**

### 2. Homepage title slightly over 60 characters
**Page:** Homepage
**Evidence:** Title is 68 characters: "The Hermetic Flight | Aerial Tarot Deck — Where Symbolism Meets Suspension" — 8 chars over guideline.
**Fix:** Trim to under 60 characters. Consider: "The Hermetic Flight | Aerial Tarot Deck" (39 chars).

### 3. Homepage meta description under 150 characters
**Page:** Homepage
**Evidence:** 131 characters. Could expand to include a keyword-rich call-to-action.

### 10. Canonical trailing slash inconsistency on Quiz page
**Page:** Quiz

### 11. Footer heading hierarchy uses H3 → H4
**Pages:** All (shared footer)
**Evidence:** Footer uses H3 ("The Hermetic Flight") then H4 ("Navigation", "Connect"). This is acceptable for footer content but technically skips from the main content's heading hierarchy.

### 12. robots.txt blocks /api/ path
**Evidence:** `Disallow: /api/` is correct for preventing crawling of API endpoints, but the `Disallow: /wp-admin/` and `Disallow: /_next/` rules reference paths that don't exist on this Astro site. These are harmless but indicate a templated robots.txt.
**Fix:** Remove non-applicable rules (`/wp-admin/`, `/_next/`) for cleanliness.

## Scoring Methodology

Each page is scored against 18-20 applicable checks (20 for blog posts, 18 for non-blog pages, excluding blog-specific checks like datePublished and word count). Each check scores: Pass = 5 points, Warn = 2 points, Fail = 0 points. Page score = (earned points / max possible points) × 100. Overall score = average of all page scores.
